###### Markdown强调语法
----------------------

### 粗体和斜体

| Markdown 语法 | 渲染效果 |
|--------------|----------|
| <code>`**粗体文本**`</code> | <strong>粗体文本</strong> |
| <code>`__粗体文本__`</code> | <strong>粗体文本</strong> |
| <code>`*斜体文本*`</code> | <em>斜体文本</em> |
| <code>`_斜体文本_`</code> | <em>斜体文本</em> |
| <code>`***粗斜体文本***`</code> | <strong><em>粗斜体文本</em></strong> |
| <code>`___粗斜体文本___`</code> | <strong><em>粗斜体文本</em></strong> |


### 删除线

| Markdown 语法 | 渲染效果 |
|--------------|----------|
| <code>`~~删除的文本~~`</code> | <del>删除的文本</del> |


---
### 注意事项

1. **星号/下划线与文本之间不能有空格**
   - ✅ `**粗体**` 正确
   - ❌ `** 粗体 **` 错误（会显示为星号）

2. **兼容性**
   - `**粗体**` 和 `*斜体*` 是 **CommonMark 核心规范**，所有编辑器都支持
   - `~~删除线~~` 不在 CommonMark 核心规范中，是 **GFM 扩展语法**
   - 现代主流编辑器（GitHub、Typora、VS Code 等）都支持删除线扩展

3. **嵌套使用**
   ```markdown
   这是***粗体和斜体***的组合。
   这是**粗体中包含*斜体*文本**。
   ```

   渲染效果： 
   
   这是***粗体和斜体***的组合。
   这是**粗体中包含*斜体*文本**。
