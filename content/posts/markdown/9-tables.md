###### Markdown表格语法
----------------------

> **注意**：表格语法不在原始 Markdown 规范中，是 **GitHub Flavored Markdown (GFM)** 扩展。但现代主流编辑器都支持。

### 基本表格

使用 `|` 分隔列，使用 `-` 分隔表头和数据。

#### 示例

```markdown
| 列1 | 列2 | 列3 |
|-----|-----|-----|
| 数据1 | 数据2 | 数据3 |
| 数据4 | 数据5 | 数据6 |
```

#### 渲染效果

| 列1 | 列2 | 列3 |
|-----|-----|-----|
| 数据1 | 数据2 | 数据3 |
| 数据4 | 数据5 | 数据6 |

**简写形式**（可以省略首尾的 `|`）：

```markdown
列1 | 列2 | 列3
-----|-----|-----
数据1 | 数据2 | 数据3
数据4 | 数据5 | 数据6
```

---
### 对齐方式

在表头分隔行中使用 `:` 控制对齐。

| 语法 | 说明 |
|------|------|
| `:-----` | 左对齐 |
| `:----:` | 居中对齐 |
| `-----:` | 右对齐 |

#### 示例

```markdown
| 左对齐 | 居中 | 右对齐 |
|:-------|:----:|-------:|
| 内容1 | 内容2 | 内容3 |
| 数据A | 数据B | 数据C |
```

#### 渲染效果

| 左对齐 | 居中 | 右对齐 |
|:-------|:----:|-------:|
| 内容1 | 内容2 | 内容3 |
| 数据A | 数据B | 数据C |

### 表格中的格式

表格单元格内可以包含**链接、代码、粗体、斜体**等。

#### 示例

```markdown
| 语法 | 示例 | 说明 |
|------|------|------|
| 粗体 | `**text**` | **粗体效果** |
| 斜体 | `*text*` | *斜体效果* |
| 链接 | `[text](url)` | [GitHub](https://github.com) |
| 代码 | `` `code` `` | `print("hello")` |
```

#### 渲染效果

| 语法 | 示例 | 说明 |
|------|------|------|
| 粗体 | `**text**` | **粗体效果** |
| 斜体 | `*text*` | *斜体效果* |
| 链接 | `[text](url)` | [GitHub](https://github.com) |
| 代码 | `` `code` `` | `print("hello")` |

---
### 表格内的换行

标准 Markdown 表格不支持直接换行，但可以使用 HTML `<br>` 标签。

#### 示例

```markdown
| 列1 | 列2 |
|-----|-----|
| 第一行<br>第二行 | 单行文本 |
```

#### 渲染效果

| 列1 | 列2 |
|-----|-----|
| 第一行<br>第二行 | 单行文本 |

---
### 空单元格

#### 示例

```markdown
| 列1 | 列2 | 列3 |
|-----|-----|-----|
| 数据1 | | 数据3 |
| | 数据5 | |
```

#### 渲染效果

| 列1 | 列2 | 列3 |
|-----|-----|-----|
| 数据1 | | 数据3 |
| | 数据5 | |

---
### 转义表格

如果不想显示为表格，可以使用反斜杠转义 `|`。

#### 示例

```markdown
这不是表格 \| 这是一个竖线
```

#### 渲染效果

这不是表格 \| 这是一个竖线

---
### 注意事项

1. **兼容性**
   - 表格是 **GFM 扩展语法**
   - 原始 Markdown 规范不支持
   - 现代编辑器（GitHub、Typora、VS Code 等）都支持

2. **格式要求**
   - 每行列数必须相同
   - 表头分隔行的 `-` 数量至少 3 个
   - `|` 和单元格内容之间可以有空格（推荐加空格以提高可读性）

3. **限制**
   - 不支持单元格合并（rowspan/colspan）
   - 不支持表格内嵌套代码块（可以使用行内代码）
   - 多行文本需要使用 `<br>` 标签

4. **最佳实践**
   - 列之间保持一致的空格
   - 表格内容简洁，避免过宽
   - 对齐方式根据数据类型选择（数字右对齐、文本左对齐）
